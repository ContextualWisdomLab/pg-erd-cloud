🔒 Sentinel: [CRITICAL] Snowflake DSN에 대한 누락된 SSRF 검증 수정 및 100% 테스트 커버리지 달성

🎯 작업 내용:
- `_parse_snowflake_dsn`에 `app.pg_introspect.dsn_guard`의 `_validated_ip_hosts`를 도입하여 Server-Side Request Forgery (SSRF) 취약점을 수정했습니다.
- SSRF 검증을 통해 Snowflake DSN에 내부망 IP(localhost, 127.0.0.1, 169.254.169.254 등)가 입력되는 것을 차단합니다.
- Snowflake 연결 시 SNI/SSL 인증에 문제가 없도록 실제 IP 주소가 아닌 원본 `parsed.hostname`을 그대로 사용하도록 구현했습니다.
- 관련 테스트 모듈의 테스트 커버리지를 100%로 올리기 위해 `test_snowflake_introspect_coverage.py`를 추가했습니다.
- 사용되지 않는 코드(`if ctype is None: continue` 부분 등)를 제거하여 가독성을 높이고 커버리지를 개선했습니다.

⚠️ 위험:
기존에는 외부 입력값(DSN)을 검증하지 않아 공격자가 임의의 내부 IP 및 포트로 요청을 유도할 수 있었습니다(SSRF 공격 가능성). 이는 내부 네트워크의 민감한 리소스 노출이나 서비스 거부(DoS) 공격으로 이어질 수 있습니다.

🛡️ 해결 방법:
Postgres 연결에서 사용하는 것과 동일하게 `_validated_ip_hosts`를 비동기적으로 호출하여 DNS 기반 IP 검증 로직을 추가했습니다.
이를 위해 `_parse_snowflake_dsn`과 `introspect_snowflake`를 비동기식으로 리팩토링하고, `asyncio.to_thread`를 사용하여 실제 연결 블로킹 작업을 별도의 쓰레드에서 안전하게 처리합니다.

✅ 검증:
- `backend/tests/test_snowflake_dsn_ssrf.py`에 SSRF 차단 테스트 케이스 추가 완료.
- `backend/tests/test_snowflake_introspect_coverage.py`를 통한 모든 Edge Case 테스트 추가로 Snowflake Introspect 모듈의 커버리지 100% 달성.
- 모든 테스트(`pytest`) 통과 확인.
