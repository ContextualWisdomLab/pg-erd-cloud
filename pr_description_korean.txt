🛡️ Sentinel: [CRITICAL] Fix Authentication Bypass via X-Dev-User Header

🚨 심각도: CRITICAL
💡 취약점: 프론트엔드와 백엔드에 `X-Dev-User` 헤더를 통해 사용자를 지정할 수 있는 개발용 기능의 잔재가 남아있었습니다. 백엔드의 직접적인 취약점은 해결되었으나, 프론트엔드에서 여전히 `localStorage`에 `devUser`를 설정하고 API 요청 시 `X-Dev-User` 헤더를 전송하여 잘못된 설정 및 클라이언트 측 문제가 남아있었으며, 백엔드의 CORS `allow_headers` 설정에도 `X-Dev-User`가 여전히 허용되어 있었습니다.
🎯 영향: 백엔드 엔드포인트가 잘못 구성된 경우 우회나 스푸핑에 악용될 수 있으며, CORS에 해당 헤더를 남겨두는 것은 앱이 이 헤더를 수용할 의도가 있음을 나타냅니다.
🔧 해결 방법: `frontend/src/api.ts`와 `frontend/src/App.tsx`에서 `devUser` 상태 및 로컬 스토리지 로직을 모두 제거했습니다. `backend/app/main.py`의 `CORS_ALLOW_HEADERS`에서 `X-Dev-User` 헤더를 제거했습니다.
✅ 검증: 업데이트된 헤더 매커니즘으로 API 호출을 테스트하였고 모든 자동화 테스트를 성공적으로 통과했습니다.
