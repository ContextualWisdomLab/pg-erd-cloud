🎯 **무엇을:**
`backend/app/sanitize.py`에 있는 `strip_nul` 함수에 대한 테스트가 누락되어 있었습니다. 이를 해결하기 위해 `strip_nul` 함수가 문자열이 아닌 입력값(`None`, 숫자, 리스트 등)을 받았을 때 오류를 발생시키지 않고 원본 값을 그대로 반환하도록 타입 체크(`isinstance(value, str)`) 로직을 추가하여 더욱 견고하게 수정했습니다. 또한, 이 함수에 대한 테스트 코드를 새롭게 작성했습니다.

📊 **테스트 커버리지:**
`backend/tests/test_sanitize.py`에 새롭게 추가된 `test_strip_nul` 함수는 다음과 같은 시나리오들을 검증합니다:
- 문자열 중간에 널 바이트(`\x00`)가 포함된 경우 정상적으로 널 바이트가 제거되는지 확인
- 널 바이트(`\x00`) 단일 문자로만 이루어진 문자열 입력 시 빈 문자열이 반환되는지 확인
- 널 바이트가 전혀 포함되지 않은 일반 문자열 입력 시 원본 문자열이 그대로 반환되는지 확인
- 문자열이 아닌 입력(`None`, 숫자 `123`, 리스트 `["\x00"]`)이 전달될 경우, 추가된 예외 처리 로직에 의해 변경 없이 원본 값이 그대로 반환되는지 확인 (Edge Cases)

✨ **결과:**
- 누락되었던 `strip_nul` 함수에 대한 테스트 커버리지를 성공적으로 확보했습니다.
- 입력값 타입 검증을 추가하여 다양한 타입의 입력에 대해서도 애플리케이션이 더욱 안정적으로 동작하게 되었습니다 (코드 안정성 향상).
- 백엔드 전체 테스트 스위트 및 프론트엔드 검증 도구(타입 체크 등)를 실행하여 아무런 회귀(Regression) 문제가 없음을 확인했습니다.
