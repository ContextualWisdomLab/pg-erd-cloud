## 2023-10-27 - [Sentinel: SSRF in Snowflake DSN parsing]
**Vulnerability:** Snowflake DSN (Database Source Name) parsing logic relied on `str.endswith(".okta.com")` to validate the `authenticator` URL for Okta integration. An attacker could bypass this by using `urllib.parse` quirks such as `https://malicious.com%5C.okta.com` or `https://malicious.com#.okta.com`, which evaluate true for `.endswith(".okta.com")` while potentially making the underlying HTTP request resolve to an attacker-controlled server.
**Learning:** `urllib.parse`'s interpretation of hostnames is often more lenient than the underlying HTTP library's (like `requests` or `httpx`), leading to Server-Side Request Forgery (SSRF). Using simple string matching like `.endswith` to validate hostnames from a parsed URL is dangerous.
**Prevention:** Always use strict regular expressions (`re.fullmatch`) to enforce allowed character sets (e.g., alphanumeric and hyphens) for subdomains before validating the target domain for external URLs.
